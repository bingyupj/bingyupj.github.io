Hugo安装并发布到Github

Posted on Feb 23, 2018

Hugo 是什么？

The world’s fastest framework for building websites.

官方说是世界上最快的建立静态网站的工具。

相类似的还有hexo,是node js 写的，hugo 是用 go 写的。

以前玩过hexo,现在还是用下hugo吧。

一：下载

官网下载地址

1）windows 环境

直接从 [releases地址](https://github.com/gohugoio/hugo/releases) 上下载相应的安装包。

下好后，是个hugo.exe文件，把该目录直接丢到环境变量里面去，方便操作。

2） mac 环境

用 brew install hugo
或者同样的从上面地址下载即可。

查看版本 ：hugo version

二 生成站点

hugo new site blog

创建的站点文件目录说明：

|- archetypes ：存放default.md，头文件格式

|- content ：content目录存放博客文章（.markdown/.md文件）

|- data ：存放自定义模版，导入的toml文件（或json，yaml）

|- layouts ：layouts目录存放的是网站的模板文件

|- static ：static目录存放js/css/img等静态资源

|- config.toml ：config.toml是网站的配置文件

当前网站是没有任何内容的，需要下载个主题跑起来才有内容。

三 下载主题

Hugo官方主题 、Hugo官方主题GitHub

从官网上选好了一个主题如：https://themes.gohugo.io/vienna/, 
根据指引进行安装，cd到 themes 目录下：

git clone https://github.com/keichi/vienna 配置好 config.toml 内容。

该主题主要包含以下内容：

|- archetypes ：存放default.md，头文件格式

|- layouts ：主题模板文件

|- static ：静态资源

|- theme.toml ：主题配置文件

四 启动

hugo server -t vienna 

( hugo server -t vienna -w -D )

五 发表文章

hugo new post/test.md

可以用文本文件或markdown编辑器打开文件 post/test.md ，并修改相应内容。

注意：默认创建的是草稿类型，需要将draft值改为false才能看到页面。

六 生成静态站点

在blog目录下执行：

hugo -t=vienna --baseUrl="https://bingyupj.github.io/"

所有静态页面都会生成到 public 目录下，生成静态网站后并push到你的GitHub Pages上，
就能得到一个在线的个人网站了。

[我的Github Page](bingyupj.github.io/)： 


番外：怎么创建 github page? 不需要了。
官网已经写的很清晰了，[点击查看](https://pages.github.com/)

Enjoy Hugo ！
